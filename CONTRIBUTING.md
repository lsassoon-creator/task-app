# Contributing to Task App

Thank you for your interest in contributing to Task App! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/task-app.git`
3. Install dependencies: `npm install`
4. Set up your environment variables (see `.env.example`)
5. Create a new branch: `git checkout -b feature/your-feature-name`

## Development Setup

### Prerequisites

- Node.js 20 or higher
- npm or yarn
- Supabase account (for local development)

### Running the Development Server

```bash
npm run dev
```

### Running Tests

```bash
npm test
```

### Linting

```bash
npm run lint
```

## Code Style

- Follow the existing code style
- Use TypeScript for type safety
- Write meaningful commit messages
- Add comments for complex logic
- Follow the ESLint configuration

## Pull Request Process

1. Update the CHANGELOG.md with your changes
2. Ensure all tests pass
3. Ensure linting passes
4. Update documentation if needed
5. Create a pull request with a clear description

## Commit Message Guidelines

- Use clear, descriptive commit messages
- Reference issue numbers when applicable
- Follow conventional commit format when possible

## Questions?

Feel free to open an issue for any questions or concerns.
